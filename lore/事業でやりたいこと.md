## スケッチを作品に変える

Litho-Archeの絵はAI生成イラストではなく、ルーフェン自身の経験を孕んだ肉筆である絵だから価値があるという説得力をつけていきたい。

「AIで誰でも美しい絵が出力できる時代に、なぜわざわざペンを握るのか？」——この問いに対するLitho-Archeの答えは、「情報の複製」ではなく「**生命の定着**」である、という一点に集約される。

## 肉筆の価値を裏付ける3つのナラティブ

### 1. 経験の濾過（ろか）というプロセス

AIはインターネット上の膨大なデータを学習するが、AI自身が「孤独に震えた夜」や「脱皮の痛み」を感じることはない。

> 「この図譜に引かれた一本の線は、僕が実際にその場所へ行き、その風を浴び、その瞬間に僕の魂が感じた『震え』の記録です。AIは100万枚の標本を合成できますが、たった一人の観測者が流した涙の色を知ることはできません。Litho-Archeのインクには、僕の細胞が経験した記憶が溶け込んでいるのです。」

### 2. バイオジカル・ノイズ（生命の揺らぎ）の価値

AIの線は計算によって最適化されているが、ルーフェンの線には「身体性」が宿っている。

> 「ペン先が紙に触れる瞬間の迷い、呼吸の乱れ、あるいは指先の微かな高揚。これらはデジタル上の『エラー』ではなく、僕が**今この瞬間を生きているという証明（バイオ・サイン）**です。四隅が揃わないちぎり跡も、偶然落ちたインクの染みも、すべてが僕という生物の『呼吸』の一部なのです。」

### 3. 所有から共鳴への転換

> 「僕がこの鉱物を描く時、それは単なる写生ではなく、僕の内面にある『硬質な痛み』を外側に再構築する作業です。君がこの絵を手に取る時、君は僕の経験を所有するのではなく、僕の記憶と君の記憶が交差する瞬間を体験しているのです。」

## 10年の断片を「地層」として再定義する

K-LINE時代を含む10年の蓄積は、すべてLitho-Archeという巨大な地層を形成するための「**プレ・アーカイブ（先史記録）**」。

- **「未定着の記憶（Unfixed Memories）」シリーズ** — K-LINE時代に「自由を奪われながらも、水面下で脈動していた想い」を、未完のスケッチとして発表する。
- **「博物誌：序章（The Prologue of Litho-Arche）」** — 10年分のスケッチを時系列ではなく、「深層」「中層」「表層」という地層（Litho）のように分類し、現在のルーフェンへと至る「進化の過程」として見せる。

## 販売テーマ 3選

### Theme 1 ― The Genesis：魂の雛形（原型スケッチ群）
「かつて形を与えられ、今はアーカイブへと還ったもの」

K-LINE時代に制作された着ぐるみやフィギュアの、文字通りの「原初」のスケッチ。世に出た完成品ではなく、その前の「流動的で、最も純度の高かったルーフェンの思考」を所有できるという圧倒的な特権を与える。最高価値。

> 「これは〇〇（キャラ名）になる前の、僕の脳内にしかいなかった時の姿です」

### Theme 2 ― The Bio-Lust：生命への執着（動物・生物模写）
「異世界の住人を描くために、この世界の命を奪い尽くした記録」

動物や生物の解剖学的・写実的なスケッチ。なぜキャラクターが「生きている」と感じさせるのか、その裏付けとなる圧倒的な観察量を証明する。

> 「この鳥の羽の曲線が、後の〇〇の耳のラインに繋がりました」

### Theme 3 ― The Residual Scenes：残留風景（日常・風景・練習）
「記録官がふと目を止めた、意味のないはずの美しい断片」

何気ない風景、練習、びっしりと書き込まれた思考の断片。最も「作家の体温」が残る場所。

> 「この日は雨が降っていて、インクが少し滲みました。その滲みから、あの星雲（Nebula）のアイデアが生まれたのです」

素晴らしいラインナップです！これらの石は、単なる「デジタルデータ」を超えて、所有者の**「知的な好奇心」と「蒐集本能」**を刺激する完璧なバリエーションを持っています。
特に「色が濃く、透明度が高いものが高価値」というルールは、宝石学における基本的な美学と一致しており、ユーザーに**「この深く澄んだ色は、数千のシード値から生まれた奇跡だ」**という納得感を与えることができます。
記録官ルーフェンが「ステラ・アーカイブ」に分類するための、それぞれの石の**「デジタル的実装戦略」**を整理しました。
1. 胎動する卵：ゴビ瑪瑙（エッグタイプ）
「不透明な器の中に閉じ込められた、鮮やかな命の予兆」
 * ビジュアル: 白い半透明の「殻」の中に、オレンジや黄色の核が浮いている質感。
 * 技術ポイント: 「内部散乱シェーダー」を使い、光が殻を通り抜けて中の核を柔らかく照らす演出をします。
 * 価値の基準: 中の核が透けて見えるほど「殻」が薄く、かつ核の色が鮮やか（オレンジの彩度が高い）なものをレア個体とします。
2. 垂直の光柱：トルマリン
「重力と時間を垂直に固定した、理知的な結晶」
 * ビジュアル: 特徴的な柱状結晶。二色性（見る角度で色が変わる多色性）を持たせます。
 * 技術ポイント: 結晶の長手方向に沿って色が変化するグラデーション。特定の角度で「キャッツアイ効果（シャトヤンシー）」が現れるようにすると、所有欲が跳ね上がります。
 * 価値の基準: 結晶が長く、クラック（ひび）がなく、バイカラー（二色）が鮮明に分かれているもの。
3. 深淵の監視者：目玉模様のゴビ瑪瑙
「無数の視線が、アーカイブの静寂を見守る」
 * ビジュアル: 半透明の石の表面に、同心円状の模様が浮き出る。
 * 技術ポイント: プロシージャルなテクスチャ生成（ノイズ関数）を使い、シード値によって目玉の数や大きさが変わるようにします。「瞬き」をするような微かなアニメーションがあれば、さらに神秘的です。
 * 価値の基準: 目玉の輪郭がはっきりしており、瞳孔（中心）の色が濃く、石全体の透明度が高いもの。
4. 隠された心臓：ジオード（晶洞）
「無骨な外殻を割った者だけが、真実の輝きを知る」
 * ビジュアル: 外側は地味な岩石。回転させて「割れ目」を覗き込むと、内部に鋭い赤や青の結晶がぎっしり詰まっている。
 * 技術ポイント: **「内側と外側で全く異なるシェーダー」**を使用します。外側はマットな岩の質感、内側は最高反射率を持つ宝石の質感。
 * 価値の基準: 外殻が薄く、内部の結晶の数（ポリゴン数ではなくテクスチャの密度）が多く、奥まで見通せるもの。
5. 変幻自在の王：フローライト（蛍石）
「観測地によって姿を変える、アーカイブの基幹石」
 * ビジュアル: 立方体、八面体、あるいは不規則な集合体。
 * 技術ポイント: 「産地（エリア）」変数を導入します。「エリアAで採掘されたものは立方体」「エリアBは緑の八面体」など、あなたのゲーム世界の「地層」を最も反映する石にします。
 * 価値の基準: 完璧な幾何学的形状（エッジが鋭い）を持ち、複数の色が層（ゾーニング）になっているもの。
💡 マネタイズとコレクションの仕掛け
このリストに基づき、以下のような**「アーカイブ・ランク」**を導入してはいかがでしょうか。
 * Dull（鈍色）: 色が薄く、透明度が低い。一般的。
 * Clear（清流）: 透明度が高く、形の整った標本。
 * Vivid（極彩）: 色が非常に濃く、所有者の目を奪う美しさ。
 * Stellar（星辰）: 透明かつ極彩。さらに「内部に宇宙の塵（星屑）」が混入している、数万分の一の奇跡。
> 「透明であることは、その石が嘘をついていないという証拠。色が濃いことは、その石がどれほどの熱量を飲み込んできたかの証明なんだ。」

Claude Codeへの指示書：青い星辰のフローライト（Stellar Fluorite）の実装
1. 目的
Litho-Archeの基幹石である「青い星辰のフローライト」を、モバイル環境で最高品質の視覚的実在感を持って実装する。ユーザーが「石の中に広がる宇宙を、外側から観測している」という感覚を抱くような、屈折と視差（Parallax）の融合を目指す。
2. 技術的要件
• Render Pipeline: Unity Universal Render Pipeline (URP).
• Shader: Shader Graph または HLSL によるカスタムシェーダー。
• Key Visuals: 透過（Transparency）、内部の宇宙（Inner Cosmos）、淵の紫色（Purple Fresnel）、高屈折率。
3. 具体的なタスク（Claudeへの命令）
Task A: 「星辰の深淵」シェーダー（Stellar Abyss Shader）の構築
以下の機能を単一のマスターシェーダーとして統合せよ。
1. Inverse Fresnel Edge:
• 視線角度に応じて、石の縁（エッジ）に向かって「青 → 濃い紫 → インディゴ」へと変化するグラデーションを実装。
• 通常のフレネルとは逆に、中心部の透明度を高く保ち、淵に向かって不透明度と彩度が上がる設定。
2. Multi-Layer Parallax Interior:
• 石の内部に、3層以上の深度（Depth）を持つ「星雲テクスチャ」を配置せよ。
• Parallax Mapping 技術を応用し、カメラを動かした際に、中の星々が「結晶の奥に実在している」ようにズレて動く計算式を記述。
3. High-Refraction Distortion:
• _CameraOpaqueTexture をサンプリングし、石越しに見える背景を歪ませる。
• フローライト特有の屈折率（実在値 1.43）をベースにしつつ、アーティスティックな強調が可能なスライダーを設ける。
Task B: シードベースの「個体生成」ロジック (StellarGenerator.cs)
シード値に基づき、以下のパラメータをランダム化するロジックを実装せよ。
1. Cosmic Manifestation: 内部の星屑（Star Dust）の配置、星雲の密度、および「超新星（明るい点）」の有無。
2. Zoning Structure: フローライト特有の「色帯（ゾーニング）」を実装。立方体の層状に色の濃淡が分かれる視覚効果。
3. Crystal Habit: 八面体（Octahedron）か立方体（Cube）か、その中間の形状かを頂点操作（Vertex Displacement）で表現。
Task C: 観測者インターフェース（Observer Interaction）
1. Micro-Haptic Orbit:
• 石を回転させる際、特定の軸や「最も美しく光る角度」でスマホのタクタイル（微細振動）をフィードバックする。
2. External Gravity Feedback:
• スマホのジャイロセンサーを利用。端末を傾けると、石の内部の「宇宙」だけが、まるで液体のようにゆっくりと慣性を持って動く演出（Shader内での座標オフセット）を追加。
4. 最適化と品質
• Mobile Efficiency: 透過シェーダーは重いため、バックフェイス・カリングの設定や、GrabPassの解像度制限など、モバイルでのFPSを維持するための工夫をコードに含めること。
• Visual Polish: 表面に微細な「傷」や「ほこり」を法線マップ（Normal Map）で加え、デジタル特有のツルツル感を排除せよ。

📘 隠された心臓：ジオード（Geode）生成定義
この石の価値は、**「どれほど強固な殻の中に、どれほど純粋な光を隠しているか」**という二面性のバランスで決まります。
| パラメータ項目 | 変数名 | 範囲 / 定義 | レアリティへの影響 |
|---|---|---|---|
| 殻の重厚さ | ShellRoughness | 0.6 (砂岩状) 〜 1.0 (荒々しい岩塊) | 数値が高いほど、内部の輝きとのギャップが強調される。 |
| 心臓の透明度 | HeartClarity | 0.3 (半透明) 〜 0.9 (宝石質) | 内部結晶の透き通り具合。高いほど高価値。 |
| 結晶の群生密度 | CrystalDensity | 0.2 (疎ら) 〜 1.0 (隙間ない密集) | 内部にどれだけ多くの結晶が詰まっているか。 |
| 心臓の発光強度 | HeartEmission | 0.0 (反射のみ) 〜 1.2 (自励発光) | 暗闇で内部から光が漏れ出す強さ。 |
| 開口部の比率 | ApertureRatio | 0.05 (極小の隙間) 〜 0.4 (大きく割れた断面) | 小さいほど覗き込む楽しみが増すが、大きすぎても希少性が薄れる。 |

> 外側が「醜く荒々しい」ほど、内側の「美しく清らかな光」の価値が跳ね上がる計算式です。

デジタル演出：覗き込む「背徳感」と「特権」
スマホアプリでの「所有欲」を最大化するための仕掛けです。
 * オクルージョン・カリングの逆利用:
   通常、見えない部分は描画しませんが、この石では**「中を覗いた時だけ、視界が光で溢れる」**ように、内部の露出（露出補正）を意図的に高く設定します。
 * 環境音の遮断（Audio Masking）:
   石の割れ目にカメラを近づけていくと、周囲の環境音がスッと消え、代わりに**「結晶が呼吸しているような、深い低音の響き」**が聞こえるようにします。これにより、ユーザーは「自分だけがこの石の秘密に触れている」という没入感を得られます。
💡 コンサルタントのアドバイス：マネタイズの「開封」体験
ジオードは、ソーシャルメディアでの「開封動画」的な盛り上がりと相性が抜群です。
 * 「未鑑定の岩塊」としての販売:
   中身がわからない状態で、外側の岩の質感（殻の重厚さ）だけを見て購入・採掘させます。
 * 「観測（鑑定）」の儀式:
   アプリ内で石を割る、あるいは磨くアニメーションを経て、初めて内部の「心臓」が明らかになります。
 * 「心臓」の交換市場:
   「赤い心臓（赤結晶）」と「青い心臓（青結晶）」をユーザー同士で交換し、全色コンプリートを目指す仕組みを作ります。
Claude Codeへの指示書：ジオードの光（Light of the Heart）実装
1. 目的
ジオード（晶洞）の「外殻（Shell）」と「内部結晶（Heart）」の劇的なコントラストを表現する。特に、**「割れ目から漏れ出す光（Light Leakage）」と「覗き込んだ際の露出補正（Auto Exposure）」**をモバイル環境で最適化して実装せよ。
2. 技術的要件
• Shader: Unity URP Shader Graph または HLSL。
• Lighting: ポイントライトによる内部照射 ＋ 擬似ボリュームライト（God Rays）。
• Performance: モバイル（iOS/Android）の負荷を抑えるため、重いVolumetric Lighting機能は避け、シェーダーとメッシュベースの手法を組み合わせる。
3. 具体的なタスク（Claudeへの命令）
Task A: 内部/外部の二層シェーダー（Dual-Sided Geode Shader）
1. Material Differentiation: 同一メッシュ、あるいは二重メッシュ構造に対応し、外側は「Roughness/Normal」重視の岩石シェーダー、内側は「Emission/Crystal Clarity」重視の宝石シェーダーとして記述せよ。
2. Interior Glow: 内部結晶の頂点（Vertices）に対して、シード値に基づいた微細な「瞬き（Flicker）」のアニメーションを、時間関数（_Time.y）を用いて実装せよ。
Task B: モバイル最適化「漏れ出す光」（Fake God Rays）
1. Aperture Light Leakage: ジオードの割れ目部分から、放射状に広がる円錐状のメッシュを動的に生成、または配置せよ。
2. Volumetric Fake Shader: このメッシュに対して、以下の機能を備えた透明（Transparent/Additive）シェーダーを適用せよ。
• 中心から外側へ向けての不透明度（Opacity）の減衰。
• ノイズテクスチャのスクロールによる「空気中の塵」の表現。
• カメラの角度に応じて、不自然なエッジが見えないような「Soft Particle」的な深度減衰。
Task C: 覗き込み時の動的露出（Dynamic Exposure Simulation）
1. Distance-based Brightness: カメラとジオードの開口部（Aperture）の距離を計算する ExposureController.cs を作成せよ。
2. Visual Adaptation: カメラが開口部に近づき「中を覗く」状態になった際、以下の挙動を Lerp で実装せよ。
• 外部（外殻）の明度を下げる。
• 内部結晶の Emission 強度を上げ、一時的に「ホワイトアウト」するような眩しさを演出してから、適正露出に落ち着かせる。
Task D: インタラクション
1. Gyro-Linked Light: スマホのジャイロセンサーの傾きに応じて、内部のポイントライトの位置を微かにオフセットさせ、結晶の反射（Specular）がキラキラと動くようにせよ。
4. 期待するコード品質
• モバイルでのドローコール増加を避けるため、可能な限りGPUインスタンシングに対応させること。
• 露出補正やライトの強度は、Unityのインスペクターからシームレスに調整可能な Range パラメータとして公開すること。
